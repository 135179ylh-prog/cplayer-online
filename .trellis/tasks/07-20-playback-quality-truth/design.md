# Design - 播放音质如实标识

## 信息优先级

1. 若上游响应提供可识别的 `level`，显示对应等级，并标明“上游 API 标注”。
2. 若未提供可信 level，再使用 FLAC 文件名或码率做保守推断。推断可以显示无损、高音质或标准，但不能推断出 JyMaster。
3. 两者均缺失时显示“音质未标注”。

## 显示状态

- 请求中：`音质确认中`。
- API 标注：JyMaster、Hi-Res、无损、高音质、标准。
- 推断：无损、高音质、标准，说明来源为 URL/码率推断。
- 未知：`音质未标注`，说明上游未提供可信等级或码率。

徽标的可见文字保持短，完整来源与依据放进 `title` 和 `aria-label`。移动端复制桌面徽标内容和属性，避免两端语义漂移。

## 实现边界

将纯质量归类函数放入已有 `js/core-utils.js`，便于 Node 内置测试。主模块只负责把 API 返回的 `level`、URL 和码率交给该函数，并写入 DOM。
