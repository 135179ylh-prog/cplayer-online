# PRD - 播放音质如实标识

## 背景

播放器默认向上游请求 `jymaster`，但不能保证每首歌都有该档音源。当前界面在请求开始就显示 `JyMaster`，并且当上游没有返回可识别码率时可能保留该文案，容易让用户把请求偏好误解为实际音源质量。

## 目标

- 明确区分“上游 API 标注的音质”和“根据 URL/码率推断的音质”。
- 对无明确质量信息的歌曲显示“音质未标注”，绝不显示默认的 JyMaster。
- 保持默认优先请求 `jymaster` 的行为，不改变可播放性和失败跳过策略。
- 桌面和移动端显示相同、真实的质量状态。

## 接受标准

1. 新歌加载时质量徽标显示“音质确认中”，而不是 JyMaster。
2. API 明确返回 `level` 时，使用该等级并说明它来自上游标注。
3. 没有可信 `level` 时，最多依据可解析的 URL/码率展示“无损”“高音质”或“标准”；缺失信息显示“音质未标注”。
4. 320 kbps 不再错误显示为 Hi-Res；除非 API 明确标注，不显示 JyMaster。
5. 质量映射具有自动化测试，桌面/移动徽标同步并具有可读说明。

## 非目标

- 寻找或接入未经验证的备用音乐 API。
- 强制拒绝非母带歌曲或自动多档降级重试。
- 在浏览器中尝试解析音频比特流来验证服务端元数据。
