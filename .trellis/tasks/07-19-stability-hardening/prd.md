# PRD - 播放器稳定性加固

## 目标

在不增加账号、协作等多人功能的前提下，提高个人长期使用时的弱网反馈、队列恢复、移动端同步、PWA 更新和本地存储可靠性。

## 需求

- API 请求不能无限等待；搜索、歌曲地址、歌词和在线歌单应在超时后进入已有错误处理。
- 桌面和移动端连续发起搜索时，较早返回的旧结果不得覆盖新结果。
- 当前播放列表写入 IndexedDB 时应避免并发旧快照覆盖；页面隐藏或退出前应立即触发待保存队列。
- 空播放列表是有效状态，清空后刷新不能重新加载以前的在线歌单。
- JSON 导入应规范化歌曲数据、同步桌面与移动列表，并作为当前队列持久化。
- 移动端所有队列入口应共用同一个 `MobileUIManager` 实例并即时刷新。
- Service Worker 应优先获取新页面、断网时回退缓存；核心缓存失败时保留旧版本；封面缓存数量必须有上限。
- 默认部署不再请求不存在的 `playlist.js`；外部歌曲标题不得作为未转义 HTML 注入。

## 验收标准

- [x] 桌面和移动搜索均通过慢请求/快请求竞态回归。
- [x] 队列在 `pagehide` 冲刷后可从 IndexedDB 读到最新状态。
- [x] 带旧歌单 ID 的空队列刷新后仍为 0 首。
- [x] 移动搜索“加入”后当前播放列表立即显示歌曲且无错误。
- [x] PWA 缓存包含全部核心资源，断开本地服务器后仍可打开 `v31` 页面。
- [x] 封面缓存最多保留 160 条。
- [x] 静态语法、功能断言和 Git 空白检查通过。
