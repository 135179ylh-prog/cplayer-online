# 完成剩余播放与备份功能

## Goal

连续完成最近播放列表、歌单导出/导入备份和播放失败自动跳过，并覆盖持久化、错误处理与移动端验证。

## Requirements

- 最近播放只记录真正开始播放的歌曲，不记录仅加载成功或自动播放被浏览器拦截的歌曲。
- 最近播放按最近时间倒序、同一歌曲去重、最多保留 50 首；支持重播、清空并在刷新后保留。
- “音乐资料库”在桌面和移动端均可打开，通过“我的歌单 / 最近播放”标签切换，提供完整空态和计数。
- 导出所有自建歌单为带格式标识和版本号的 UTF-8 JSON 文件，不导出当前队列或最近播放。
- 导入前完整校验文件大小、格式版本、歌单与歌曲字段；为导入歌单生成新 id，保留现有歌单，并在单个 IndexedDB 事务中原子写入。
- 导入失败不得产生半写入歌单；重复名称生成可辨识后缀，并显示成功数量或明确错误。
- 获取播放地址、无有效 URL、媒体加载或播放失败时自动跳到下一首；同一故障链每个队列项最多尝试一次，全部失败时停止，不能无限循环。
- 浏览器 `NotAllowedError` 自动播放策略拦截不视为歌曲失败，不触发自动跳过。
- 自动跳过与播放结束都遵循统一的四种模式值：`sequence / repeat_one / repeat_all / shuffle`；兼容迁移旧值 `single / random`。
- 保持后台播放使用原生 `<audio>` 路径，不引入 WebAudio 路由或新依赖。

## Acceptance Criteria

- [x] 成功开始播放后最近列表新增记录；重复播放移动到顶部，列表最多 50 首，刷新后仍存在。
- [x] 最近列表可重播歌曲、清空历史，空列表显示明确空态。
- [x] 桌面和移动端均能打开音乐资料库，标签与按钮不重叠。
- [x] 导出 JSON 包含版本、导出时间和全部自建歌单；可在空浏览器中导入并还原名称、歌曲及顺序。
- [x] 非 JSON、错误版本、超大文件或非法歌曲数据会被拒绝，且 IndexedDB 不产生部分记录。
- [x] API/媒体失败会按模式跳过；连续坏歌不会重复尝试，全部失败后停止并提示。
- [x] `NotAllowedError` 保留当前歌曲等待用户手动播放，不跳过。
- [x] 四种播放模式只切换一次并正确持久化，旧 `random/single` 值会迁移。
- [x] JavaScript/Service Worker 语法、结构断言和真实浏览器关键路径验证通过。

- 现有未跟踪补丁脚本仅作为历史研究，不直接执行或覆盖 `index.html`。
- 按自用场景优化操作密度：新建歌单支持回车、资料库记住当前标签、桌面与窄屏操作区分别排版，不加入账号、分享或协作功能。
