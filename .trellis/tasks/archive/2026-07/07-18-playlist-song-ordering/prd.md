# 自建歌单歌曲排序

## Goal

补齐自建歌单管理详情，支持歌曲上移、下移和顺序持久化，并覆盖移动端交互。

## Requirements

- 在“我的歌单”中点击“管理”后打开当前歌单的歌曲详情。
- 每首歌曲提供播放、上移、下移和移除操作；上移/下移交换歌曲在 `songs` 数组中的位置。
- 第一首歌曲的上移按钮、最后一首歌曲的下移按钮必须禁用，不能产生越界写入。
- 排序和移除完成后写回现有 IndexedDB `playlists` object store，并刷新详情及歌单歌曲计数。
- 关闭详情或刷新页面后再次打开同一歌单，顺序必须保持。
- 覆盖加载中、空歌单、歌单不存在和保存失败等基本反馈；移动端按钮需要可触达并带有可访问名称。
- 排序只改变自建歌单记录，不应隐式清空或重排当前播放队列；点击播放操作时才加载队列。

## Acceptance Criteria

- [x] “管理”按钮可以打开详情并显示歌单名称、歌曲数量和原顺序。
- [x] 对中间歌曲执行上移、下移后，界面顺序和 IndexedDB 中的 `songs` 顺序一致。
- [x] 首尾边界按钮禁用；单曲和空歌单不会抛出未处理异常。
- [x] 移除歌曲后数量、列表和持久化数据同步更新。
- [x] 重新打开详情/刷新页面后顺序仍然正确。
- [x] 静态脚本语法检查、项目特性检查和关键浏览器路径验证通过。

## Notes

- 现有应用是单页静态 PWA，业务代码集中在 `index.html`，数据使用 IndexedDB。
- 保持已有播放列表、歌单创建/加入/播放/删除行为兼容。
