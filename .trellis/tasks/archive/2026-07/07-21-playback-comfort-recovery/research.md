# Research - 播放舒适度与状态恢复

## 现有能力

- `current_queue` 已保存歌曲、`currentIndex`、播放模式和时间戳，并在页面启动时恢复。
- `timeupdate` 目前只更新界面和歌词，没有持久化播放秒数。
- `online` / `offline` 事件已有一次性 Toast。
- 网络请求已有 15 秒超时、一次重试和服务器错误重试判断。
- 播放失败会按播放模式寻找未失败过的下一首，避免死循环。

## 缺口

- 刷新后只能恢复“选中了哪首”，不能继续具体进度。
- 失败提示没有区分断网、服务异常和无音源。
- 项目没有睡眠定时。

## 决策

沿用现有单页结构做小范围接入，同时把可测试的判断逻辑放入已有 `core-utils.js`。不在这一轮启动大规模文件拆分，避免把用户体验改进与架构迁移混在一起。

## Claude 状态

受保护的 Claude Code 一次性只读分析在达到最大轮数后退出，没有生成结论，因此不作为实现依据。诊断保存在 `research/claude-analysis.md`，本轮由 Codex 独立实现和验证。

## 浏览器验证补充

- 恢复流程按歌曲 ID 在持久化队列中重新定位，不依赖旧索引；刷新后仅提示并等待用户点击播放。
- 睡眠定时使用绝对截止时间，刷新后可以继续计算剩余时间；取消和到点都会清理本地记录。
- 桌面与移动设置窗口均能容纳新增控件，截图保存在本任务的 `research/` 目录。
- 自审发现音源时长变化会影响恢复位置；已用 `getSafePlaybackResumeTime` 统一保存与应用阶段的“距结尾至少 5 秒”规则，并补充回归测试。
