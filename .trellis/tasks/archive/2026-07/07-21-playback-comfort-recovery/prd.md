# PRD - 播放舒适度与状态恢复

## 目标

让播放器关闭或刷新后可以继续上次的歌曲与进度，并提供简单可靠的睡眠定时；当播放失败时，用用户能理解的原因提示，而不是统一显示“播放失败”。

## 已确认事实

- 当前队列、当前索引和播放模式已经保存在 IndexedDB，但没有保存具体播放秒数。
- 页面已有在线/离线 Toast，也已有网络超时、有限重试和播放失败自动跳过。
- 浏览器通常不允许页面刷新后自动播放，因此恢复后必须等待用户点击播放。
- 设置窗口适合放置睡眠定时，不需要新增复杂页面。

## 需求

1. 保存当前歌曲、队列索引、播放秒数、总时长和更新时间；播放中最多每 5 秒写入一次，暂停、切到后台和关闭页面时立即保存。
2. 页面重开后恢复到同一首歌和上次进度，但不自动播放；用户点击播放后继续。
3. 无效、过期、接近歌曲结尾或已不在队列中的恢复记录应被忽略或清理。
4. 设置中提供关闭、15、30、45、60、90、120 分钟的睡眠定时，并显示剩余时间。
5. 睡眠定时跨刷新保留；到点后暂停播放、清理定时并明确提示。
6. 区分断网、请求超时/服务异常和歌曲无可用音源三类播放失败提示，同时保留有界自动跳过。

## 验收标准

- [x] 播放超过 5 秒后刷新，队列仍在，点击播放会跳回保存进度附近。
- [x] 恢复过程不会绕过浏览器自动播放限制，也不会在后台自行发声。
- [x] 清空队列会清除恢复记录；移除当前歌曲后不会恢复已删除歌曲。
- [x] 睡眠定时的设置、取消、倒计时和到点暂停在桌面及移动布局中可用。
- [x] 刷新页面后，未到期的睡眠定时仍继续计时。
- [x] 自动化测试覆盖恢复记录校验、睡眠时间计算和失败原因分类。
- [x] 构建、语法、特性检查和真实浏览器主路径验证通过。

## 非目标

- 不做账号、云同步或跨设备续播。
- 不承诺浏览器被系统彻底关闭后仍能在后台准点唤醒执行定时器。
- 不接入未经验证的备用音乐接口。
