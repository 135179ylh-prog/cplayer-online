# PRD - 搜索失败重试与恢复

## 目标

当第三方搜索服务临时失败或设备离线时，让用户保留当前搜索上下文并能直接重试，而不是只看到一行错误文字后重新操作。

## 已确认事实

- 桌面关键词搜索失败后只显示“搜索服务暂不可用”，没有后续操作。
- 移动关键词搜索失败后只显示“搜索出错”，没有后续操作。
- 桌面和移动搜索已经分别使用请求序号避免旧响应覆盖新结果。
- 搜索依赖第三方 `api.chksz.top`，网络和服务短时失败是已记录的真实场景。

## 需求

1. 桌面与移动关键词搜索失败时显示明确原因：离线显示“当前已离线”，其他失败显示“搜索服务暂不可用”。
2. 失败状态提供可见的“重试”按钮，点击后使用原查询词重新搜索。
3. 重试不得清空搜索输入、关闭桌面侧栏或移动底部面板。
4. 桌面与移动共用失败状态渲染逻辑，避免文案、无障碍属性和按钮行为再次分叉。
5. 保留现有请求序号保护；较早请求的成功或失败响应不得覆盖较新的搜索状态。

## 验收标准

- [x] 桌面搜索失败后显示原因和“重试”，输入词与搜索面板保持不变；重试成功后正常显示结果。
- [x] 移动搜索失败后显示原因和“重试”，输入词与底部面板保持不变；重试成功后正常显示结果。
- [x] 离线状态使用“当前已离线”，在线服务失败使用“搜索服务暂不可用”。
- [x] 失败状态具有可读状态语义，重试按钮具有包含查询词的无障碍名称。
- [x] 自动化检查、语法检查和真实浏览器桌面/移动回归通过。

## 非目标

- 不修改纯数字歌曲 ID 的直接加载与播放流程。
- 不自动无限重试，也不新增备用音乐接口。
- 不缓存或持久化搜索结果。
