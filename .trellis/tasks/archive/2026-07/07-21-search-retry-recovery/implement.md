# Implementation - 搜索失败重试与恢复

1. 搜索现有桌面与移动异常分支，确认请求序号和面板状态边界。
2. 增加共享失败状态渲染函数，并接入桌面关键词搜索。
3. 接入移动关键词搜索，保持底部面板与输入状态。
4. 增加静态特性检查，验证两个入口都使用共享函数。
5. 运行构建、测试、语法、特性和差异检查。
6. 用 Playwright 模拟“首次失败、点击重试后成功”，分别验证桌面和移动布局。

## 风险与回退点

- 风险集中在 `index.html` 的两个异常分支；不改 API、IndexedDB 或队列格式。
- 若共享渲染函数异常，可恢复原静态错误文字，不影响成功搜索路径。
