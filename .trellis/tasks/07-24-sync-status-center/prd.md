# 同步状态中心

## Goal

让账号与歌单云同步状态在桌面和手机上可见、可理解、可重试，并准确展示待同步、冲突、最近成功时间和错误，不改变本地优先边界。

## Requirements

- 桌面与手机的设置入口都要显示同步健康状态；用户不打开设置也能知道
  当前是已同步、待同步、冲突或错误。
- 设置中的账号卡升级为状态中心，统一显示：状态名称、说明、真实待同步
  数量、真实冲突数量、最近一次成功同步时间、最近错误与可执行的重试入口。
- 待同步数量必须来自 IndexedDB `cloud_outbox`，不能用内存猜测；已退出时
  仍提示本机存在的待同步项目，但只有登录对应账号后才能发送。
- 冲突数量必须与当前仍存在的冲突一致。重新检查成功后，已经消失的旧
  冲突不能继续显示；每个冲突仍必须显式选择“使用本机”或“使用云端”。
- 最近成功时间只在本轮同步确认没有待同步项目、没有冲突时更新，并按
  账号保存在本机。错误、离线或部分完成不能伪造成功时间。
- 手动同步若仍有待同步项目或冲突，不得提示“同步完成”。错误时保留本机
  数据并提供重试；重试成功后清除错误展示。
- 未配置或未登录时，播放、本地歌单、备份等功能保持完整可用。状态中心
  不得阻塞应用 ready 信号或播放。
- ChKSz API 密钥、API 地址、播放队列、最近播放、播放进度和设备设置继续
  只存本机，不进入 Supabase、云端歌单 payload 或备份。
- 沿用现有 Supabase Free、GitHub Pages、IndexedDB v5 和既有 RPC；本阶段
  不新增云表或数据库迁移。

## Acceptance Criteria

- [x] 桌面 `1280x800` 和手机 `390x844` 的设置入口具有可读状态摘要，状态
  中心无溢出、按钮可操作且通过现有无障碍检查。
- [x] 登录同步成功后显示“已同步”、待同步 0、冲突 0 和最近成功“刚刚”；
  刷新后仍能从本机恢复该账号的最近成功时间。
- [x] 离线修改真实写入 outbox 后显示待同步 1；联网发送完成后变为 0，且
  持久化歌单与云端 mock 均一致。
- [x] 冲突显示真实数量与当前序号；选择本机或云端后数量减少，未选择前
  不覆盖任何一份歌单。
- [x] 注入网络错误时显示错误详情和“重试同步”；重试成功后错误消失并
  进入已同步状态。
- [x] 未配置、未登录和退出登录场景仍可本地使用；遗留 outbox 不会以其他
  账号身份发送。
- [x] 单元、静态、桌面/手机账号同步测试、完整响应式测试与
  `npm run verify` 全部通过；Trellis 验证文档记录准确计数。
- [ ] 本阶段以独立提交推送 `main`，Pages 工作流成功，线上状态中心加载并
  完成只读冒烟验证后，才能进入“真实跨设备验收”。

## Out Of Scope

- 真实双设备数据验收（下一里程碑）。
- 删除回收站、历史版本、冲突内容差异和自助恢复（后续里程碑）。
- 云端保存播放记录、播放进度、API 配置或设备设置。
