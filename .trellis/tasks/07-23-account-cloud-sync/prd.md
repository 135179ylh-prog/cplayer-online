# 账号与云同步基础

## Goal

为 CPlayer 增加安全、可选的用户账号和云端数据同步，让用户换浏览器或换设备后仍能找回自己的歌单，同时保留现有无需账号即可播放和离线打开的能力。

## Background

- 当前应用部署在 GitHub Pages，是纯静态网页，没有自有服务器或用户数据库。
- 当前队列和自建歌单保存在浏览器 IndexedDB `CPlayer5DB` 中；最近播放、播放进度、播放模式、睡眠定时、API 地址和 API 密钥等保存在 `localStorage` 中。账号同步在本任务中将数据库从 v4 增量升级到 v5。
- IndexedDB 当前版本为 v4。`current_queue` 与 `user_pl_*` 是用户数据，其他远程歌单和图片记录属于可丢弃缓存。
- 现有导出备份只包含自建歌单，不包含当前队列、最近播放、播放进度、睡眠定时或 API 设置。
- ChKSz API 密钥目前只保存在当前浏览器。账号系统不得把该密钥上传到云端、写入源码、测试、文档或 Git 历史。
- 推送 `main` 会自动部署线上站点；规划和本地实现不得擅自推送、部署或创建真实线上资源。

## Product Decisions

- 账号是可选增强。未登录用户继续使用现有全部本地功能。
- 第一版使用邮箱和密码注册/登录，并提供邮箱找回密码。
- 第一版只同步自建歌单及歌曲顺序。当前队列、最近播放、播放进度、播放模式、睡眠定时、沉浸模式和 ChKSz API 设置继续只保存在本机。
- 首次同步按稳定歌单 ID 合并：两边不同的歌单都保留；同一歌单只在一边变化时采用变化的一边；两边都变化时暂停该歌单并让用户明确选择“使用本机”或“使用云端”。
- 采用 Supabase Auth + Postgres。网页只包含允许公开的项目 URL 和 publishable/anon key；服务端管理密钥不进入客户端或 Git。

## Requirements

### ACCT-1: 账号与会话

- 提供邮箱密码注册、登录、退出和忘记/重置密码的完整入口与反馈。
- 登录状态应跨刷新保持；会话失效、断网和服务异常必须给出用户能理解的提示。
- 用户只能读取和修改自己的云端数据，不能通过修改浏览器请求访问他人的数据。

### ACCT-2: 本地优先与向后兼容

- 未登录时，现有搜索、播放、队列、本地歌单、备份和离线启动继续可用。
- 登录和同步不能阻塞播放器启动或正在进行的播放。
- 云服务不可用时继续使用本地数据，并明确显示“尚未同步”，恢复网络后可重试。

### ACCT-3: 云端同步

- 第一版至少同步用户明确拥有的自建歌单和歌单内歌曲顺序。
- 首次登录时必须先处理当前设备已有歌单，不能静默覆盖本地或云端数据。
- 每次本地或云端变更都需要更新时间或版本信息，用于检测冲突和避免旧数据覆盖新数据。
- 删除行为必须有清晰语义，并能在其他已登录设备上正确反映。
- 本地歌单写入和待同步记录必须在同一个 IndexedDB 事务中完成，避免页面中断后漏同步。
- 自动同步只处理当前登录账号拥有或本次明确接管的记录，不能把另一个账号的本地待同步记录上传到当前账号。

### ACCT-4: 隐私与安全

- 密码不得由播放器代码自行保存或处理明文，必须交给成熟的身份认证服务。
- ChKSz API 密钥、API 地址、睡眠定时和设备相关设置默认只留在本机，不进入云端同步。
- 云端数据库必须启用逐用户访问控制，并提供注销账号及删除云端个人数据的路径。
- 客户端允许包含服务商规定可公开的项目标识，但任何具有后台管理权限的密钥都不得进入网页产物或 Git。

### ACCT-5: 质量与发布边界

- 账号、离线、本地已有数据、同步冲突和权限边界必须有确定性的单元或浏览器测试；测试只模拟网络边界，不访问真实用户账号。
- 保留现有 `npm run verify` 全量门禁，并为数据库结构和访问策略增加可重复检查。
- 实际创建云项目、配置域名、推送和部署前必须再次取得用户确认，并提供回退方案。

## Acceptance Criteria

- [x] 未登录用户无需注册即可继续使用当前全部本地功能。
- [x] 用户可完成注册、登录、退出和凭据恢复，刷新后登录状态正确。
- [x] 登录用户在两台模拟设备间能同步自建歌单、歌曲和顺序。
- [x] 首次登录遇到本地已有歌单时，不会静默丢失或覆盖任一侧数据。
- [x] 断网时本地编辑仍可保存，并显示待同步状态；恢复后能安全重试。
- [x] 两台设备同时修改同一歌单时，不会无提示地用旧版本覆盖新版本。
- [x] 用户无法读取或修改其他账号的歌单；自动化测试覆盖此权限边界。
- [x] 注销账号会删除该账号的云端个人数据，但不会误删其他账号的数据。
- [x] ChKSz API 密钥不会出现在云端同步载荷、构建产物、测试固定值或 Git 历史中。
- [x] 完整质量门禁通过；未获得明确许可前不推送、不部署、不创建真实线上资源。

## Out Of Scope For The First Version

- 社交关注、公开主页、好友、评论、共享编辑和公开歌单广场。
- 音乐文件上传、缓存音频跨设备同步或绕过第三方音乐服务限制。
- 管理员后台、付费订阅和复杂多租户组织系统。
- 实时协作编辑；第一版只需可靠检测冲突并让用户明确选择。
- 手机验证码、第三方社交登录、自定义邮件服务器和付费能力。
