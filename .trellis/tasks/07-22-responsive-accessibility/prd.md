# PRD - 响应式与无障碍一致性

## 目标

让 CPlayer 5 的核心界面在窄屏手机、常见手机、宽折叠屏和桌面端都能稳定显示，并让用户可以只用触摸、键盘或读屏完成播放、切换视图、打开面板和使用弹窗。

## 已确认事实

- 当前自动浏览器回归只有桌面 `1280x800` 和手机 `390x844`；`355px` 窄屏与宽折叠屏没有固定自动化项目。
- `355x800` 实测没有整页横向滚动，但底部 8 个操作挤在同一行，部分按钮被压缩到不足 44px，文字变成狭窄竖排。
- 关闭的桌面侧栏与移动底部面板仍可进入浏览器可访问树，键盘可落到不可见控件。
- 设置弹窗打开后，焦点仍停在弹窗后方的设置按钮；Esc 不会关闭弹窗。
- 音乐资料库和歌单详情各自监听 Esc。详情叠加在资料库上时，同一次按键可能关闭两层。
- 桌面和移动播放进度目前是仅响应点击的 `div`，没有滑块语义和键盘调节能力。
- 移动封面/歌词切换主要依赖滑动或点击唱片，缺少明确的键盘入口；非当前页面仍会暴露给读屏。

## 需求

### R1 弹窗层级与焦点

- 设置、加入歌单、音乐资料库、歌单详情和欢迎弹窗使用同一套覆盖层管理规则。
- 打开弹窗后，焦点进入弹窗；背景内容不可被点击、聚焦或读屏访问。
- Tab 与 Shift+Tab 只在最上层弹窗内循环。
- Esc 只关闭最上层允许关闭的弹窗；歌单详情关闭后资料库保持打开。
- 关闭后焦点返回打开入口；动画弹窗在真正隐藏后再恢复背景与焦点。

### R2 侧栏、底部面板与标签页

- 关闭的桌面侧栏和移动底部面板必须 `aria-hidden` 且不可聚焦；打开时同步入口的 `aria-expanded`。
- 面板打开后焦点进入当前标签；Esc 关闭并把焦点返回入口。
- 桌面、移动和资料库标签页具备 `tablist` / `tab` / `tabpanel` 关系，并支持左右方向键切换。
- 当前未展示的标签面板不可被键盘或读屏访问。

### R3 播放器核心控件

- 桌面与移动播放进度拥有滑块语义、当前时间说明和键盘调节；左右方向键按固定步长跳转，Home/End 跳到开头/结尾。
- 音量滑块有可读名称，音量弹层状态同步到入口。
- 移动端提供 44px 的封面/歌词切换按钮；非当前视图对读屏隐藏。
- 可点击的歌曲主区域必须有原生按钮或等价键盘语义，行内“加入、歌单、移除”等操作保持独立。

### R4 响应式与触控

- 移动底部控制区分成主播放行与次要操作行，不再把播放模式、清空、歌单等 8 个动作挤在一行。
- 手机可见的主要按钮、标签页和歌曲行操作目标不小于 `44x44` CSS 像素。
- `355x800`、`390x844`、宽折叠屏 `440x707` 和桌面 `1280x800` 均不得出现整页横向滚动、控件重叠或按钮文字被挤成竖排。
- 布局变化不改变播放、搜索、队列、歌单、设置和 PWA 更新的数据契约。

### R5 自动化质量门禁

- 引入 `@axe-core/playwright`，在真实 Chromium 中扫描核心静态界面和打开后的弹窗/面板。
- 新增专用窄屏与宽折叠屏 Playwright 项目；现有业务用例仍只运行桌面和常见手机，避免无意义倍增。
- 浏览器测试覆盖焦点进入/返回、Tab 圈定、嵌套弹窗 Esc、关闭面板不可聚焦、进度键盘操作、触控尺寸和横向溢出。
- 自动测试不得调用真实 ChKSz 服务，不在生产代码加入测试分支。

## 验收标准

- [x] 五类弹窗遵守统一焦点和背景隔离规则；嵌套详情按一次 Esc 只关闭详情。
- [x] 桌面侧栏与移动底部面板关闭时不在可操作树中，打开/关闭状态和入口属性同步。
- [x] 播放进度可用键盘调节，封面/歌词有明确按钮，标签页可用方向键切换。
- [x] 动态队列与搜索歌曲的“播放”主动作可通过键盘触发，行内其他按钮保持独立。
- [x] 四种目标视口无整页横向滚动、无控件重叠；移动主要触控目标均至少 44px。
- [x] Axe 自动扫描无 critical/serious 违规，新增交互回归在四种视口通过。
- [x] `npm run verify` 八层门禁和 `git diff --check` 通过。
- [x] Service Worker 缓存版本、Trellis 规范、任务验证记录与生产资源同步。

## 范围外

- 不改第三方 ChKSz API、密钥、播放质量或存储格式。
- 不做视觉品牌重设计，不更换现有流体背景与字体图标体系。
- 不推送、不部署；本任务只做本地提交与归档。
