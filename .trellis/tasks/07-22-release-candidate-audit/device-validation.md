## 发布前必须

以下项目默认均为“未执行”，只有在真实设备完成后才能改为通过。

- [ ] 竖屏打开播放器，首屏无横向滚动，底部按钮不重叠，文字不竖排。
- [ ] 展开到宽屏形态后，封面、进度、主控制区和底部操作仍完整可见。
- [ ] 搜索并播放一首可用歌曲；切到其他应用 60 秒，音乐持续播放。
- [ ] 锁屏 60 秒后音乐持续；锁屏界面显示当前歌曲和播放/暂停、上一首、下一首。
- [ ] 使用系统媒体通知暂停和继续，播放器页面状态同步。
- [ ] 关闭屏幕再点亮，播放进度没有回跳到 0，也没有重复开始同一首歌。
- [ ] 播放时从 Wi-Fi 切到移动网络，失败时有明确反馈；恢复网络后可重新操作。
- [ ] 播放队列中加入至少 2 首歌，刷新页面，队列顺序和当前歌曲仍在。
- [ ] 打开更新提示后选择“稍后刷新”，音乐和页面不被打断；再次更新时选择“刷新”，队列和进度仍可恢复。
- [ ] 导出一份歌单备份，导入有效备份成功；导入损坏 JSON 不删除已有歌单。

## TalkBack 与操作

- [ ] 开启 TalkBack，设置按钮读作“打开设置”，关闭后焦点回到入口。
- [ ] 播放列表/搜索标签会读出选中状态，左右方向或滑动浏览不会进入隐藏面板。
- [ ] 打开音乐资料库和歌单详情，返回键/Esc 每次只关闭最上层，焦点回到原按钮。
- [ ] 播放进度读出当前时间/总时长；调节后歌曲跳到对应位置。
- [ ] 封面/歌词切换按钮读出当前动作，隐藏视图不会被重复朗读。
- [ ] 搜索歌曲的“播放”“加入”“歌单”是三个独立动作，不会误触发两个操作。

## 建议补充

- [ ] 蓝牙耳机播放/暂停、上一首、下一首按键可用。
- [ ] 开启系统省电模式后重复 5 分钟后台播放。
- [ ] 来电或其他应用抢占音频后，返回播放器能继续由用户控制。
- [ ] 无网络启动已访问过的 PWA，应用壳和本地歌单可查看，在线功能显示离线反馈。
- [ ] 清理浏览器进程后重新打开 PWA，队列、歌单和设置仍在。

## 失败记录格式

每个失败至少记录：

- 使用的设备、系统、浏览器版本和横竖/折叠状态；
- 操作步骤与预期结果；
- 实际结果、屏幕录制或截图；
- 是否能稳定复现；
- 是否只发生在省电、锁屏、蓝牙或网络切换状态。
