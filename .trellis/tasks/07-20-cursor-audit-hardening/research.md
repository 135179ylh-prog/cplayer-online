# Research - Cursor 审计核查

## 已核实

- `index.html` 约 334KB，运行时 Tailwind 文件约 407KB，属于真实的性能/维护债务，但一次性迁移会扩大回归面。
- 主脚本使用 `<script type="module">`；只有少量兼容 API 显式挂到 `window`，因此“全部变量都在全局”不成立。
- 监听器主要在单页初始化阶段注册，当前没有路由反复重建；`removeEventListener` 为 0 不能单独证明泄漏。
- 外部歌曲文字已在主要路径使用 `textContent` 或 `escapeHtml`；不能仅凭 `innerHTML` 总数认定 XSS。
- API 超时已有 15 秒，但基础地址重复且没有有限重试/网络状态反馈。
- 主桌面封面和移动封面缺少显式尺寸/异步解码；动态列表封面需要补固定属性。
- 根目录确有已提交的临时修补/检查脚本，应移入 `scripts/legacy/`。

## 决策依据

本轮优先改动用户可感知的稳定性和可用性，保留已上线播放、歌单和 Service Worker 行为。构建链、类型系统和全量模块化作为后续独立任务，避免把审计整改变成不可验证的大重构。

## Claude peer

受保护的 Claude 只读终端能够启动并通过 readiness probe；短审阅已开始读取任务和改动，但在限定窗口内没有返回要求的完成标记，因此不作为审批证据。其途中指出的未跟踪根目录脚本已核对：它们不会进入本轮提交，且已由 `.gitignore` 排除；已提交的历史脚本已经移入 `scripts/legacy/`。
