# PRD - Cursor 审计后的低风险加固

## 背景

Cursor 对当前播放器提出了架构、性能、安全、可访问性和工程卫生方面的审计意见。本任务只落地能够从现有代码直接验证、且不会破坏稳定播放流程的改进；一次性迁移 Vite、Tailwind 构建链或 TypeScript 不在本轮范围内。

## 目标

- 让键盘和屏幕阅读器能够识别主要的图标按钮、装饰画布和移动封面。
- 给封面和动态列表图片提供稳定尺寸与异步解码，减少布局抖动。
- 在断网、恢复联网和 API 临时失败时给出明确反馈，并使用有限重试避免瞬时故障直接中断操作。
- 清理根目录已提交的临时调试脚本，把本地生成物加入忽略规则。
- 用轻量、无需第三方依赖的测试覆盖网络重试和核心数据规范化逻辑。

## 接受标准

1. 生产页面不再把缺少标签的主要 icon-only 控件暴露给辅助技术；装饰 canvas 标记为隐藏。
2. 主桌面封面、移动封面以及动态列表封面具备 `width`/`height` 或等价稳定尺寸，且使用异步解码/懒加载。
3. 网络状态变化会通过现有 Toast 通知；请求对超时、网络错误和 5xx 最多进行有限退避重试，4xx 不盲目重试。
4. API 基础地址只保留一个配置入口，不引入未经验证的备用域名。
5. 调试脚本不再散落在仓库根目录；`__pycache__`、`*.pyc`、`.playwright-cli` 等本地产物被忽略。
6. 自动化验证可在干净 Node/Python 环境运行，并记录在 `verification.md`。

## 暂缓项

- 拆分 334KB 单文件、引入 Vite/Tailwind 构建链、全量 TypeScript/ESLint/Prettier。
- 为单页初始化监听器设计完整销毁生命周期。
- 因 `innerHTML` 数量本身而进行全量 DOM 重写；现有外部文本路径继续使用 `textContent`/`escapeHtml`。
- 自动生成 Service Worker hash 版本；本轮若不修改 SW 核心策略则不人为升级缓存名。
