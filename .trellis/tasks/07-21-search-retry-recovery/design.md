# Design - 搜索失败重试与恢复

## 共享错误状态

在主脚本中增加 `renderSearchRecoveryState(container, options)`，由桌面 `searchSongs` 和移动 `MobileUIManager.handleSearch` 的关键词搜索异常分支共同调用。

输入：结果容器、查询词、重试回调和紧凑模式。函数只负责安全创建状态文字与按钮，不读取搜索输入，也不发起请求。

输出：容器内包含带 `role="status"` 的失败状态、原因文字和“重试”按钮；按钮通过闭包调用原入口，查询词不写入 HTML 字符串。

## 行为边界

- 原入口继续负责查询规范化、加载状态、请求序号和成功结果渲染。
- `navigator.onLine === false` 时显示离线文案；其余异常显示服务不可用。
- 重试按钮不主动关闭面板、不清空输入，也不绕过现有请求竞态保护。
- 桌面使用常规间距，移动使用紧凑字号；文案与无障碍语义保持一致。

## 兼容与回退

- 仅修改异常分支；正常搜索、空结果、纯数字 ID 和收藏流程保持不变。
- 删除共享函数调用即可恢复原异常展示，数据层和 Service Worker 无迁移。
