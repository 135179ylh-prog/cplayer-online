# PRD - 搜索与歌单操作体验修复

## 目标

让自用场景下的“搜索添加、当前播放、歌单管理”形成连续且容易理解的操作流程。

## 需求

- 从搜索结果收藏歌曲到自建歌单后，关闭选择弹窗仍保留搜索面板和已有结果，不要求重新打开侧栏。
- 移动端“当前播放”歌曲行的歌单按钮必须真正打开收藏流程，并用文字说明用途。
- 搜索结果中加入当前播放列表与加入自建歌单的动作使用明确文案，不再只显示“+列表”。
- 音乐资料库中的播放、管理、删除动作要有可见文字和稳定触控目标；管理动作打开歌单详情与排序界面。
- 桌面侧栏和移动底部面板的外部点击关闭逻辑不得误伤正在使用的弹窗。

## 验收标准

- [x] 搜索结果收藏成功后，搜索面板仍打开，搜索结果仍存在；下一首歌曲可直接继续收藏。
- [x] 移动“当前播放”中的“歌单”按钮打开“加入歌单”弹窗，选择歌单后歌曲写入目标歌单。
- [x] 资料库“管理”按钮可打开详情，空歌单和有歌曲歌单均可进入。
- [x] 按钮文案能区分“加入当前播放列表（不立即播放）”与“收藏到歌单”。
- [x] 桌面和移动端弹窗交互不触发侧栏/底部面板误关闭。
