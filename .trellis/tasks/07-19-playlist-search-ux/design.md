# Design - 搜索与歌单操作体验修复

## 事件边界

页面有两个“点击外部关闭”监听器：桌面统一侧栏和移动底部播放面板。加入歌单弹窗、音乐资料库、歌单详情、设置和欢迎弹窗都属于独立的覆盖层。统一通过 `isOverlayInteractionTarget` 判断事件是否来自覆盖层，覆盖层内的点击不再触发侧栏关闭。

## 收藏流程

```text
搜索结果 / 当前播放歌曲
  -> openAddToPlaylistModal(song)
  -> 选择自建歌单
  -> addSongToUserPlaylist()
  -> 关闭选择弹窗
  -> 保留原搜索面板、输入内容和结果
```

收藏成功后自动收起选择弹窗，方便立即点选下一首；搜索结果本身不重新请求、不清空。重复收藏沿用现有去重逻辑。

## 歌单资料库

列表操作使用图标加文字的 44px 操作按钮：播放、管理、删除。管理按钮调用现有 `openPlaylistDetail`，详情弹窗继续复用 IndexedDB 读取、排序和移除逻辑。

## 移动端当前播放

当前播放歌曲行的 `+` 改为“歌单”按钮，调用同一个 `openAddToPlaylistModal` 收藏入口；旁边的移除动作改为“移除”，避免用户需要猜测符号含义。
